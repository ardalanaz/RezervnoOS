# راهنمای بک‌آپ خودکار — رزرونو

سیستم بک‌آپ **داخل خود بسته** است و با Docker اجرا می‌شود. نیازی به تنظیم cron دستی روی سرور نیست.

> داده‌ی مشتری و رزروها ارزشمندترین دارایی کسب‌وکار است. این سیستم خودکار از آن محافظت می‌کند.

---

## چطور کار می‌کند

یک سرویس به نام `backup` در `docker-compose.yml` هست که:
- روی زمان‌بندی تنظیم‌شده (پیش‌فرض: هر شب ۳ بامداد) با `pg_dump` بک‌آپ فشرده می‌گیرد
- فقط N بک‌آپ آخر را نگه می‌دارد (پیش‌فرض ۱۴) و قدیمی‌ها را خودکار حذف می‌کند
- بک‌آپ خالی/خراب را تشخیص می‌دهد و رد می‌کند
- اختیاری: به آبجکت‌استوریج (آروان/لیارا/S3) آپلود می‌کند

بک‌آپ‌ها در پوشه‌ی `backup/data/` روی سرور ذخیره می‌شوند.

---

## تنظیمات (در `.env`)

```bash
BACKUP_CRON=0 3 * * *      # هر شب ۳ بامداد (می‌توانی تغییر دهی)
BACKUP_KEEP=14             # چند بک‌آپ نگه داشته شود
BACKUP_ON_START=true       # بک‌آپ فوری هنگام بالا آمدن
```

نمونه‌های زمان‌بندی:
- `0 3 * * *` — هر روز ۳ بامداد
- `0 */6 * * *` — هر ۶ ساعت
- `0 2 * * 0` — هر یکشنبه ۲ بامداد

با تغییر `.env`، یک بار `docker compose up -d` بزن تا اعمال شود.

---

## دستورهای دستی

```bash
# گرفتن بک‌آپ فوری (همین الان)
docker compose exec backup /scripts/backup.sh

# دیدن لیست بک‌آپ‌ها
docker compose exec backup /scripts/list.sh

# لاگ سرویس بک‌آپ
docker compose logs -f backup
```

---

## 🔴 بازیابی (وقتی به فاجعه خوردی)

**بک‌آپی که نتوانی برگردانی، بی‌فایده است.** این را حتماً یک بار تست کن:

```bash
# بازیابی از آخرین بک‌آپ
docker compose exec backup /scripts/restore.sh

# یا از یک فایل مشخص
docker compose exec backup /scripts/restore.sh rezervno_20260115_030000.sql.gz
```

⚠️ بازیابی، داده‌ی فعلی را بازنویسی می‌کند. ۵ ثانیه فرصت لغو (Ctrl+C) می‌دهد.

---

## 💾 بک‌آپ خارج از سرور (مهم!)

بک‌آپی که فقط روی همان سرور است، اگر سرور بسوزد یا هک شود، **با آن از بین می‌رود**. حتماً به جای دیگری هم کپی کن.

### گزینه ۱: آبجکت‌استوریج (خودکار، پیشنهادی)
در `.env` پر کن (مثلاً آبجکت‌استوریج آروان یا لیارا):
```bash
S3_ENDPOINT=https://s3.ir-thr-at1.arvanstorage.ir
S3_BUCKET=rezervno-backups
S3_ACCESS_KEY=کلید-دسترسی
S3_SECRET_KEY=کلید-مخفی
```
حالا هر بک‌آپ خودکار به آبجکت‌استوریج هم آپلود می‌شود.

### گزینه ۲: کپی دستی به سیستم خودت
```bash
# از سرور به سیستم محلی
scp user@سرور:/path/rezervno-deploy/backup/data/rezervno_*.sql.gz ./
```

### گزینه ۳: rsync به سرور دوم
```bash
rsync -avz backup/data/ user@سرور-دوم:/backups/rezervno/
```

## ✅ چک‌لیست بک‌آپ امن

- [ ] سرویس backup بالاست (`docker compose ps` → backup را ببین)
- [ ] اولین بک‌آپ ساخته شده (`docker compose exec backup /scripts/list.sh`)
- [ ] **بازیابی را یک بار تست کرده‌ای** (مهم‌ترین قدم!)
- [ ] بک‌آپ خارج از سرور تنظیم شده (S3 یا کپی دستی)
- [ ] زمان‌بندی مناسب کسب‌وکارت است (شلوغی رستوران = بک‌آپ بیشتر)

---

## نکته‌ی امنیتی

بک‌آپ‌ها داده‌ی حساس مشتری (شماره موبایل، رزرو) دارند. مطمئن شو:
- پوشه‌ی `backup/data/` دسترسی محدود دارد
- اگر به آبجکت‌استوریج آپلود می‌کنی، باکت **private** باشد (نه public)
